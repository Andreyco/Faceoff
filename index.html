Faceoff

Wrapper and helper methods for Facebook SDK.
Built to be framework agnostic, but won't let you down
with framework of your choice.

Installation

Oldschool way, direct download

You can download Faceoff from GitHub the old way, copy it wherever you want and use!
Do not forget to download official Facebook SDK for PHP as well!

Cool way, via Composer

{
    require: {
        "andreyco/faceoff": "dev-master"
    }
}

Easy to use,
with helpfull methods

Usage

As said, Faceoff is built to be framework agnostic. Use it either with framework of your choice, or you dump frameworks and utilize pure PHP instead.

Pure PHP

Make sure package files are loaded. Manage configuration is loaded somehow, or configure it inline.
Speaking about pure PHP, require_once() is enough! 

// Load official Facebook SDK
require_once('facebook-php-sdk/src/facebook.php');

// Load Faceoff files
require_once('faceoff/src/Andreyco/Faceoff/SessionProviders/SessionProviderInterface.php');
require_once('faceoff/src/Andreyco/Faceoff/SessionProviders/NativeSessionProvider.php');
require_once('faceoff/src/Andreyco/Faceoff/Traits/Session.php');
require_once('faceoff/src/Andreyco/Faceoff/Faceoff.php');

$config = require_once('faceoff/src/config/config.php');
$sessionProvider = new \Andreyco\Faceoff\SessionProviders\NativeSessionProvider;
$faceoff = new \Andreyco\Faceoff\Faceoff($config, $sessionProvider);

// Use Faceoff!
if (! $faceoff->getUser()) {
    echo '<script>top.location.href="'. $faceoff->getLoginUrl() .'";</script>';
}

Laravel 4

Faceoff comes with support for Laravel 4 right out of the box!

To prevent loss of configuration after each update of Faceoff package, you should publish configuration files and edit these instead. 

php artisan config:publish andreyco/faceoff

Just open up config/app.php file, register Service provider and add an Alias 

'providers' => array(
    // ...
    'Andreyco\Faceoff\Support\Laravel\FaceoffServiceProvider',
    // ...
),

'aliases' => array(
    // ...
    'Faceoff'       => 'Andreyco\Faceoff\Support\Laravel\Facade',
    // ...
),

Just use it! 

// Use Faceoff!
if (! Faceoff::getUser()) {
    echo '<script>top.location.href="'. Faceoff::getLoginUrl() .'";</script>';
}

Documentation

Since Faceoff extends official Facebook SDK for PHP, it offers its native methods as well. You should browse official documentation or read the code to learn more about them.
Apart from these, Faceoff offers methods which provide nice interface to native functionality. 

array me ($fields)
Get data for current user.
Parameters $fields
List of fields to retrieve. Can be represented as comma separated list (string) or array of fields.
Return value array containing fields passed to the function. Array contains id key even if not requested. 
array friends ()
Get friendlist of current user.
Return value array containing friendlist of current user. 
mixed likesPage()
Check whether user already likes currently visited Facebook page.
Return value NULL if cannot determine, or BOOL otherwise. 
mixed administratesPage()
Check whether user is administrator of currently visited page.
Return value NULL if cannot determine, or BOOL otherwise. 
array fql($query)
Execute single FQL query or multiple queries.
Parameters $query
Single query represented as string or array of queries to execute.
Return value array of result data for each of passed queries.
